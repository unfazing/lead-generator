# Phase 03.1 Plan 02: Snapshot Import Workflow Summary

**First-class company snapshot review and recipe-level company import flow for people search**

## Performance

- **Duration:** 14 min
- **Completed:** 2026-03-23T15:32:57Z
- **Tasks:** 2
- **Files modified:** 13

## Accomplishments

- Added `/search/company/[snapshotId]` as a dedicated company snapshot review route with warnings, exact params, and row selection isolated from people-search controls.
- Reworked `/search/people` so the operator starts from a people recipe, chooses one or more company snapshots as import sources, and explicitly applies those companies to the recipe before running people search.
- Extended people recipes and people snapshots to persist organization import provenance, including source snapshot IDs, import mode, selected company IDs, and import timestamps.

## Decisions Made

- Company selection remains local UI state inside snapshot review surfaces until the user explicitly applies imports to a saved people recipe.
- The people recipe is the only source of truth for organization IDs at search time; people search no longer depends on transient company snapshot context.
- Provenance is normalized and deduped when saved so repeated imports do not produce unstable recipe or snapshot metadata.
